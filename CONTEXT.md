# Builders' Workbench — 领域语言 (Ubiquitous Language)

单人构建者的项目管理工作台。本文件是**唯一的规范词表**:代码、文档、界面、对话都用这里选定的词，别再各叫各的。

> 定义原则:每个词只说它**是什么**(1–2 句)，不写实现;`_Avoid_` 列出被它取代的说法。
> 这是词表,不是规格——设计事实源仍是 [`plan/06-overall-alignment.md`](plan/06-overall-alignment.md);
> 命名冲突的收敛与改名计划见 [`docs/adr/0001-ubiquitous-language.md`](docs/adr/0001-ubiquitous-language.md)。

## Language

### 人与角色

**Builder / 构建者**:
使用工作台的那个人——端到端 own 一个项目的单人系统设计者,产品的主角。全局唯一,专指人类。
_Avoid_: user、owner、开发者;不要用 "Builder" 指构建阶段的角色(那是**构建师**)。

**队友 / Teammate**:
被指派去干活的 AI 人格,有名字、模型、真实战绩与胜率。派活的对象是队友。
_Avoid_: Agent(留给运行时执行体)、AgentCard(那是队友在目录里的一行展示)、Squad、workbuddy、multica、机器人。

**五角色 / Stage Role**:
五个阶段各自的方法论人格:原型师·构建师·优化师·运营推广师·运维师 (Prototyper / Builder-role / Optimizer / Grower / Maintainer)。每个角色带该阶段的打法与开场指令(preamble),是内置方法论,不是某个具体队友。
_Avoid_: 把"五角色"和"聊天发言方 (Author)"混为一谈——后者只有人/AI 两方;构建师的英文别写成裸 "Builder"(与构建者撞)。

**发言方 / Author**:
一条对话消息出自谁——人(Builder)还是 AI(Agent)。纯 UI 归属轴,与"五角色"无关。
_Avoid_: 用 "Role" 单指这个(Role 已被五角色占用;代码里对应类型已改名为 `Author`,见 ADR)。

### 项目生命周期

**阶段 / Stage**:
项目的五个方法论阶段:**原型 → 构建 → 优化 → 运营推广 → 运维**,运维复盘回流到原型(**闭环回流**,所以是环不是流水线)。每阶段自带该问什么、什么节奏、DoD、常见坑。这是通用方法论,不随项目现编。
_Avoid_: 用裸词 "环 / Loop / operating loop" 指五阶段——"环/循环"留给工作流内部的微循环;五阶段的成环性叫"**闭环回流**"这个属性,不当名词用。step/step-N/七步向导(已演进为五阶段)。

**交棒 / Handoff**:
从当前阶段推进到下一阶段的那一次显式动作。只留痕、不拦人——清单没勾完也能交。
_Avoid_: 审批、gate(交棒不是审批门)、阶段切换。

**带险交棒 / Risky handoff**:
DoD 清单未勾完就交棒的情形——永久记下当时缺什么、留了几件活。你有权冒险,但不能无痕。
_Avoid_: 强制完成、跳过。

**DoD / 完成清单**:
一个阶段"做到什么算完"的明确勾选清单。纯人工勾选,永不派生、永不代填。
_Avoid_: Definition of Done 缩写混用时统一写 "DoD";不要叫验收标准(那偏向 Issue 验收)。

**时期 / Maturity Period**:
项目所处的成熟度期:探索期(0→1) · 扩张期(1→N) · 成熟期(Sustain)。用户创建时声明,纯信息性——只影响项目墙的精力配比展示,不驱动任何派生。
_Avoid_: 周期 / Cycle / ProjectCycle(裸词"周期/环"太容易和五阶段闭环、工作流循环撞;代码里对应类型已改名为 `MaturityPeriod`,见 ADR)、生命周期。

**就绪状态 / Readiness**:
项目本身的就绪度:运营中(Running)还是冷启动中(创建流程未完成)。是"状态",不是"阶段"。
_Avoid_: Phase / ProjectPhase(与"阶段 Stage"撞;代码里对应类型已改名为 `Readiness`,见 ADR)。

### 活与状态

**Issue / 活**:
一件要干的具体工作,派给队友去做的最小单位。"活"是口语规范词,"卡"只在指界面卡片形态时用。
_Avoid_: 任务(与"定时任务"撞)、卡片(指实体时用"活",指 UI 时才用"卡")、ticket、工单。

**Issue 状态流转**:
Backlog → Todo → InProgress → InReview → Done 的单向明路;另有 Blocked(阻塞,必填原因)、Cancelled。**Done 的唯一入边是 InReview**。
_Avoid_: 自造中间态;把 InReview 当终态。

**评审中 / InReview**:
队友把活干完后能到达的最远状态。到此为止,人还没确认。
_Avoid_: 完成、待办结。

**完成 / Done**:
活的终态,**永远由人显式点**——任何自动化都不许把活推到 Done。落 Done 的同一刻,队友战绩、产物登记、阶段吞吐同时入账,且同一件活绝不记两次。
_Avoid_: 自动完成、agent 自判完成。

**阻塞 / Blocked**:
活卡住了、停在原地,必须写明原因。干砸了如实停在这里,可重试,不假装前进。
_Avoid_: 失败(失败是"运行"的结局,不是活的状态)、挂起。

### 工作流

**工作流 / Workflow**:
一段可复用的多环节干活流程(每个环节由某个角色执行)。分**静态**(定版、有版本号与使用数)与**动态**(按阶段临时生成)两类。
_Avoid_: 流水线(会误导成"不回流")、pipeline、脚本。

**循环 / Loop**:
工作流**内部**的微循环——某个环节失败重试、迭代到达标为止(`retries` / `max_iter`)。这是"环/loop"这个词的正主:对抗式 agent 自闭环的小循环。
_Avoid_: 用 "Loop/环" 指五阶段或整个产品运作模式;operating loop。

**Agent Loop**(产品命题层):
"1 个 Builder + Agent Loop 取代 5 个专职角色"里的运作模式——人守质量门、agent 干 80% 的活。是产品主张的说法,专用于命题语境。
_Avoid_: 与"工作流内部循环"混用;把它当某个具体代码结构。

**运行 / Run**:
一次工作流或一件活的真实执行,有成败与耗时,全结局都入账(成功、失败都记)。
_Avoid_: 执行(动词可用,名词统一"运行")、跑一遍(口语)。

### 健康(难造假)

**观测 / Observation**:
一个真实数据点(来自网关日志 / CI / git / telemetry / 连接器 / 手填)。**只追加,一个观测=一个点,绝不插值**。
_Avoid_: 记录、log(观测是结构化数据点,不是日志文本)、埋点数据。

**指标 / Metric**:
由观测派生出的、可与目标比较的度量(带目标值、比较符、黄区带)。
_Avoid_: 数据、统计值;不要和"观测"混用——观测是原料,指标是产物。

**信号 / Signal**:
健康灯的三态结论 + 未知:绿(隐身)/ 黄 / 红 / **Unknown**。只能由真实数据派生,**无数据 = Unknown,绝不假装绿**;数据过期自动降级。
_Avoid_: 状态、健康值、分数;不要说"信号是绿的就是好的"以外的手动设定——信号从不手设。

**未知 / Unknown**:
没有数据时的诚实态,灰色。喂再多数、不设目标也不会变绿。它是产品的诚实底线,不是"待补"占位。
_Avoid_: 默认绿、N/A、待定。

**北极星 / North Star**:
每个项目唯一的顶层目标。可由工作台把用户回答组合成草稿,但绝不编造数字替用户答题。
_Avoid_: KPI、OKR、总目标;不要有多个"北极星"。

**引领指标 / 滞后指标 (Leading / Lagging)**:
引领指标=可控、可提前统计、能被真实采集的先行量;滞后指标=结果量。控制点 4 要求引领指标可控且难造假。
_Avoid_: 先行/结果指标混用不同词;过程指标。

**手填 / Manual**:
数据来源是人手工填的,带"手填"徽记,直到接上真实来源。可用,但如实标注、不冒充自动采集。
_Avoid_: 假数据、mock(mock 是另一回事,见下)、临时值。

### 自动化(只自动建活,绝不自动完成)

**定时任务 / Scheduled task (Cron)**:
到点自动触发的作业,针对某项目或全部项目。到点只做两件事之一:运行一个工作流,或**建一件活**。
_Avoid_: 例行(那是阶段的节奏件 Routine)、计划任务、job 混称;裸 "cron" 指代整个特性时也统一叫"定时任务"。

**Autopilot / 自动建活**:
定时任务的一种模式——到点把例行活挂上看板、指派好队友,**只建活、绝不自动跑、更不自动完成**(no-hijack)。
_Avoid_: 自动执行、自动运行、自动化完成——Autopilot 从不自己完成活。

**例行 / Routine**:
一个阶段自带的节奏件:该阶段的循环节奏 + 关注项 + 健康 feed。挂在阶段上,不是全局作业。
_Avoid_: 与"定时任务"混用;例行任务(会和 Cron 撞)。

### 复利(越用越强)

**蒸馏 / Distill**:
把一件做完的活提炼成一篇带完整做法的技能,永远记着它来自哪件活。
_Avoid_: 总结、归档、沉淀(沉淀指整个复利过程,别专指这一步)。

**技能 / Skill**:
一篇可复用的做法说明。下次干同类活时按相关度自动带上(同阶段优先,有上限),用一次记一次(uses)。
_Avoid_: 文档、知识、SOP;不要和"五角色的打法"混——那是内置方法论,技能是蒸馏产物。

**注入 / Inject**:
干活时自动把最相关的几条技能带进队友的上下文。
_Avoid_: 加载、引用、喂(口语,正式文档用"注入")。

**复利 / Compounding**:
"做过的活沉淀成方法、越用越懂你项目"这个整体属性:蒸馏→注入→uses+1 的增益链。
_Avoid_: 积累、飞轮;专指这条链时用"复利"。

**战绩 / 胜率 (Win rate)**:
队友的真实成败统计派生出的胜率,**永远派生、绝不手设**;无战绩时是"暂无",不是 0%。
_Avoid_: 评分、rating、可信度手调。

### 资产与来源

**资产目录 / Hub**:
可复用资产的浏览目录,分三类:工作流 / 技能 / 队友(`HubKind`)。是"库/市场"性质的浏览面,不是定时任务(别被 "Cron Hub" 旧说法误导)。
_Avoid_: 把 Hub 和 Cron/定时混为一体;中心、枢纽。

**业界来源 / HubSource**:
一份资产从哪来:oh-my-claudecode / ecc / 自建 / 会话内 / 已采纳(Adopted)等。
_Avoid_: 和"观测源 (SourceKind)"、"官方/自建 (LibSource)"混用同一个"来源"词——它们是不同轴。

**连接器 / Connector**:
项目对外部真实来源(git 仓库、claude CLI 等)的连接,能真同步的才叫连接,其余是登记条目、如实标未同步。
_Avoid_: 集成、插件、数据源。

**产物 / Artifact**:
队友运行真实产出的文件,分类登记(文档/代码/测试/脚本/配置/其他)。
_Avoid_: 输出、成果物、deliverable。

**mock / 流程演示**:
未接真实执行时的演示替身,产出**自我标注**【mock】。唯一目的是廉价验证管线本身,绝不冒充真实执行。
_Avoid_: 假装真实、demo 数据不标注;和"手填 (Manual)"混——手填是真人真数据,mock 是演示替身。
