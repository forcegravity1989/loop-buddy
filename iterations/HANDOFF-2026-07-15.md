## 3. 执行队列(接棒从 P1 开始,逐项有 DoD)

### P1 · Autopilot 最小化:cron 自动建 Issue 并指派(首选,~半天)

**为什么**:multica Autopilot 的 `create_issue` 模式是「方法论环自动化 × Issue 层」的天然交点;
BW 已有真实调度器(`App::tick_scheduler`,5s tick,no-hijack),只差「建 Issue」这一步。

**现状事实**(写码前自己再核一遍):`CronTask.target` 是自由文本,tick 里按
`specs.iter().find(|w| w.name == c.target)` 解析;`project_id=None` 的任务不自动触发;
`status != Normal` 不触发;`Cadence::Cron(_)` 表达式 `cron_due` 诚实返回 false。

**怎么做**:
1. `bw-store/schema.sql` + `sqlite.rs::open()`:cron_task 加列(**必须 `add_column_if_missing`**,
   见 [[bw-schema-migration-pattern]] 事故教训):
   - `mode TEXT NOT NULL DEFAULT 'run_workflow'`(旧行为字节不变)
   - `issue_stage TEXT`(StageKind 文本,mode=create_issue 时必填)
   - `issue_assignee TEXT`(agent **名字**,与按名记账同一约定;空=不指派)
2. `bw-core` `CronTask` 加对应字段(serde default);`CronMode { RunWorkflow, CreateIssue }`。
3. `tick_scheduler`:`mode == CreateIssue` 分支——不解析 workflow spec,改为
   `CreateIssue { stage, title: format!("[auto] {} · {}", c.name, 日期), desc: 引用 cron 任务与触发时刻, priority: Medium }`
   → 若 `issue_assignee` 非空,按名找 agent 并 `AssignIssue`
   → 建后状态:**推荐 `Todo`**(autopilot 建单=已承诺的工作;multica 语义里 Backlog 是抑制触发的停车场,
     自动建单进 Backlog 会让「自动化」名不副实)。仍然 **no-hijack:绝不自动 RunWorkflow**。
   → `record_cron_run` 记 Ok/Failed 与人工触发同语义。
4. Command 面:`CreateCronTask` 增 mode/stage/assignee 参数(桌面 CronHub 表单后补,先 headless)。
5. **测试**(App 级,新文件 `bw-app/tests/autopilot.rs`):
   - due 的 create_issue 任务 tick 后:Issue 存在、stage/assignee 正确、状态 Todo、cron last_run 更新;
   - 同 tick 幂等:再 tick(未到期)不重复建;
   - `issue_assignee` 指向不存在的 agent:Issue 照建、不指派、**不算失败**(诚实 0 匹配);
   - mode 缺省的旧任务行为与合并前字节不变。

**DoD**:上述测试全绿 + 全门禁;`record_fusion_round` 或新 example 里真实建一条
autopilot 任务演示一次 tick 落库;`plan/06 §5` 台账把 Autopilot 从留白挪到 ✅。

## 6. 坑清单(本轮踩过,带修法)

1. **`with_workspaces_root` 不配=静默不开仓**(条件直接不触发,连响亮降级都没有)——headless
   指挥器必须显式配(见 `record_fusion_round.rs` 的做法);要不要给「未配 root」加一条 loud 事件,
   由你判断(注意存量测试假设不开仓)。
2. **fmt 先于 commit**:`git commit` 前跑 `cargo fmt --all`,否则门禁在 commit 后打脸(本轮 amend 过一次)。
3. **合并 keep-both 后查函数闭合**:两处「HEAD 侧被截断在 match/方法中间」型断裂,编译器
   `unclosed delimiter` 直接指向断点(先例:`connector_status_text` / `list_artifacts`)。
4. **嵌套 claude 探测**:只 `env -u CLAUDECODE -u CLAUDE_CODE_SESSION_ID`,别动 `ANTHROPIC_AUTH_TOKEN`/
   `ANTHROPIC_BASE_URL`,否则假 401 误导判断。
5. **桌面残留实例**:`pgrep -fl builders-workbench`,先杀再起,否则两实例抢深链叙事。
6. **529 与 401 是两种病**:529=过载可重试;401=token 死,重试空转。判定前先 curl 直探。
7. 二进制名是 `builders-workbench` 不是 `app-desktop`。
