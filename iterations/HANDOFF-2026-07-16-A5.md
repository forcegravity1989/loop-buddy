## 2. P1 · A5-F 转移守卫 + Blocked 必填原因

### 2.1 冻结的设计决定(已定死,不要重开)

**合法转移表**(唯一事实,落在 `bw-core`):

| from ↓ | 合法去处 | 语义 |
|---|---|---|
| Backlog | Todo · InProgress · Cancelled | 激活 / 直接开工(=RunIssue 起跑边)/ 弃 |
| Todo | InProgress · Backlog · Blocked* · Cancelled | 开工 / 退回停车场 / 阻塞 / 弃 |
| InProgress | InReview · Todo · Blocked* · Cancelled | 提审 / 放回队列 / 阻塞 / 弃 |
| InReview | Done · InProgress · Blocked* · Cancelled | **人确认完成(记账边)** / 打回 / 阻塞 / 弃 |
| Blocked | Todo · InProgress · Cancelled | 解除回队列 / 解除继续干 / 弃 |
| Done | Todo · InProgress | 重开(settle-once 已兜底,不重计) |
| Cancelled | ∅ | 终态,无出边 |

其余一律非法(含 X→X 自转移、Backlog→Done 直跳、Cancelled→任何)。
**Done 只能从 InReview 进**——看板前进链不受影响(InProgress→InReview→Done 两击)。

- **守卫位置**:`IssueStatus::can_transition_to(self, to) -> bool` 加在
  [crates/bw-core/src/model.rs:1391](../crates/bw-core/src/model.rs) `is_terminal` 旁
  (纯函数,wasm 安全);**App 层执行,store 保持哑**(与现分层一致)。
- **Blocked 入口 = 新命令** `Command::BlockIssue { id: IssueId, reason: String }`
  (加在 [crates/bw-app/src/lib.rs:402](../crates/bw-app/src/lib.rs) `AssignIssue` 旁)。
  `reason.trim()` 为空 → `AppError::Invalid`(「转 Blocked 必须给出阻塞原因」)。
  来源必须 ∈ {Todo, InProgress, InReview}(即表中 Blocked* 三行;查同一张表)。
  `TransitionIssue` 对 `status == Blocked` 一律拒绝并提示走 `BlockIssue`——原因不可绕过。
- **`issue.blocked_reason` 列**(TEXT,可空):
  [crates/bw-store/src/schema.sql:343](../crates/bw-store/src/schema.sql) 的 CREATE TABLE 加列
  **且** [crates/bw-store/src/sqlite.rs:137](../crates/bw-store/src/sqlite.rs) 旁加
  `add_column_if_missing(&pool, "issue", "blocked_reason", "TEXT")`
  (存量表不吃 CREATE TABLE 新列——这是本仓库摔过的坑,见 memory/schema 纪律)。
- **不变量:离开 Blocked 即清空原因**。实现定死:store `transition_issue`
  ([sqlite.rs:1957](../crates/bw-store/src/sqlite.rs))的 UPDATE 顺手 `blocked_reason=NULL`
  (BlockIssue 之外没人能进 Blocked,所以 transition_issue 无条件清空是对的);
  新增 store 方法 `block_issue(id, reason)`:单条 UPDATE 同时置
  `status='blocked', blocked_reason=?, updated_at=?`。
- **RunIssue 起跑前置**([lib.rs:2036](../crates/bw-app/src/lib.rs)):`get_issue` 后检查
  状态 ∈ {Backlog, Todo, InProgress},否则 `AppError::Invalid`(如
  「Issue #N 处于评审中,不能直接重跑;先打回 InProgress」)。
  InProgress 属合法起点 = **重试语义**(A3 失败如实留 InProgress,重跑是唯一恢复路)。
  已是 InProgress 时**跳过**起跑转移(自转移非法);成功边 InProgress→InReview 在表内,不动。
- `TransitionIssue` 处理器([lib.rs:2432](../crates/bw-app/src/lib.rs)):`prev` 为 None →
  `NotFound`(现在是静默 UPDATE 空转);非法转移 → `AppError::Invalid`,消息带
  `#{number} {from}→{to}`。**守卫插在 store 写之前;2441 起的 Done 边沿记账一行不动**。
  被拒的命令经 kernel 现有错误面呈现(toast,[kernel.rs:198](../crates/app-desktop/src/kernel.rs) 一带),UI 侧无需新管道。

## 3. P2 · A5-H 看板交互 + 项目墙开放计数

### 3.1 冻结的设计决定

- **看板加第 6 列「阻塞」**(现 5 列在 [op.rs:554](../crates/app-desktop/src/screens/op.rs))。
  今天 Blocked 卡片不落任何列=看板隐身,这违反 bw-core 自己的注释
  (「treating Blocked as done would hide stuck work」)。Cancelled 仍不上板(终态弃活),留白如实。
- **阻塞卡片**:显示 `blocked_reason`(必非空)+ 两个解除键「解除→待办」「解除→进行中」
  (走 `TransitionIssue`,表内合法)。
- **进入阻塞**:Todo/InProgress/InReview 卡片加「⛔ 阻塞」;点击展开行内原因输入 + 确认
  (板级 signal `Option<(IssueId, String)>` 即可,同一时刻只编辑一张卡)→ `Command::BlockIssue`。
  原因为空时确认键不发命令。
- **指派**:卡片上现在的 assignee 文本([op.rs:628-638](../crates/app-desktop/src/screens/op.rs))
  升级为 `select`:选项 = 「未分配」+ hub 全员(`AgentCardVm{id,name,role}`,
  [vm.rs:651](../crates/ui/src/vm.rs);五角色实体天然在列)→ `Command::AssignIssue`(命令已存在,
  [lib.rs:2499](../crates/bw-app/src/lib.rs),纯 UI 缺口)。
- **倒退/重开/取消故意不给 UI**(plan/06 §7-H 原文;API 层有守卫+settle-once 兜底)。
  前进键(`next_issue_status`)保持唯一前进面,不加拖拽。
- **项目墙开放计数**:卡片带「开放 Issue 数」=该项目非终态件数(`!is_terminal`,
  与 A4 交棒口径**同一谓词**,[lib.rs:1704](../crates/bw-app/src/lib.rs));Blocked 算开放。
  仅 >0 时渲染小徽记(如 `⚑ N 开放`),0 不渲染(不制造噪音)。
