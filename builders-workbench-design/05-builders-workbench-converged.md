# 05 Builders Workbench · 收敛设计（三轮对抗后，愿景做实）

> 经 R1→R3 三轮 A 级红蓝对抗。R3 裁定「**还差关键一两块、补上即成**」并给了 5 条建设性 P0。本文把它们全部补进用户愿景，得到一个**能站住**的 Builders Workbench。
> **关键突破**：R2 锯断的飞轮轴，被 R3 的「**认证策略（而非验收模板）**」接回——货架装的是认证策略，真飞轮成立。

---

## §1 三轮对抗的净结果
- **R1**：v0 被拆穿（persona 空、范畴错、工厂 PPT、无闸、无分级器、过载、挑软柿子）。
- **R2**：基石 BREAK——反向拆解产不出可复用「模板」（验收**内容**领域特定、零迁移）。
- **R3**：蓝队转向后愿景**第一次长成对的形状**（两类舰员、Builder=指挥官、闭环完整），但 5 面承重墙没砌完。
- **→ 本文**：补完承重墙。

---

## §2 关键突破：货架装的是「认证策略」，不是「验收模板」（修复 R2 的 BREAK）
- **R2 是对的**：验收的**内容**（写哪条断言）领域特定、零迁移 → 「验收模板」不可复用。
- **R3 补上**：但验收的**判据/策略**可复用——Builder 认证「**这一类**验收模式我已判定充分」一次，此后命中该模式的新 Goal **免逐条认证**。
- → **货架 = 认证策略库（可复用的判断准则），不是验收内容模板。** 这：
  - **不违反 R2**（内容仍不可复用）；
  - **给了真飞轮**：认证策略越攒越多 → 新舰员边际认证成本下降（修 P0-D）；
  - **破认证瓶颈**：命中策略的新 Goal 免认证 → 认证队列收敛（修 P0-B）。
- **这是三轮挤出的真资产**：Builder 的稀缺判断被**复利成认证策略**，而非被每条 Goal 一次性消耗。（这正是人类组织扩展判断力的方式：不逐案重判，而是定策略、管例外。）

---

## §3 产品形态（全生命周期舰队，机制补全）

**① 舰队（两类舰员 + 两个环节的诚实归属）**
| 环节 | 归属 | 机制 |
|---|---|---|
| 需求开发 / 集成构建 / 运维 / 上线动作 | **执行器 Loop**（闭合） | Prompt+Workflow+Team+可终止 Goal，自主跑到验收 |
| 洞察 / 运营 | **Goal 工厂/监测器**（开放） | 常驻 Routine，不求终止；带**采纳率代理闸**（修 P0-C）|
| 需求设计（伪闭合） | **拆解** | 拆成可断言子任务（出 3 方案 + 评测矩阵）喂执行器；「拍板就它了」留给人；地图上诚实标「辅助」（修 P0-A）|
| 项目管理（元层） | **不是舰员** | 它就是工作台的**路由 + 地图**本身（修 P1-A）|

**② 信任脊柱**：每个舰员两道闸——验收闸（人认证「充分+客观+抗漂移」，但通过**认证策略**复利化）+ 副作用闸（对外/花钱/不可逆 → 人批 + kill switch + 日志）。Goal 工厂用**采纳率代理闸**过脊柱（浮出决策的 Builder 采纳率连续低于阈值 → 自动降频/静音）。

**③ 工厂（诚实定位）**：不是「自我生长永动机」，是 **标准化准入闸 + 编排样板库 + 认证策略库**。它造「壳与闸」，认知核（验收内容）仍人造——但认证被策略复利化，所以**新同类舰员近乎免认证**。这才是真实、可建、有用的飞轮。

**④ 路由引擎**：分级器只浮「需认证 / 需审批 / 校验失败 / 卡住」；超出 Builder 认证带宽则对 Goal 工厂**限流背压**（修 P0-B）。

**⑤ 生命周期地图**：两类舰员的覆盖度，需求设计诚实标「辅助」——不在图上装绿（修 P0-A）。

**⑥ Builder**：责任 = 全生命周期（ALL）；杠杆 = 认证**策略**（不是每条 Goal）+ 批副作用 + 编排舰队。

## §4 Skill→Plugin→Loop→Workbench 闭环（最终）
- Skill（能力，~50）→ Plugin（工具箱，2-3）→ **Loop=舰员**（Plugin+认证Goal+两闸+循环）→ **Workbench**=运行（路由）+生长（工厂+认证策略库）+守住（脊柱）+看清（地图）。
- 用户信息第 5 条「**汇总 = Builders Workbench**」在此坐实：汇总的**不是模板，是认证策略 + 舰队状态**——这是可复利、可站住的汇总。

---

## §5 MVP（砌地基的顺序，朝全产品收敛）
**第一块砖**：信任原子——两道闸 + 人认证验收 schema + **认证策略机制** + 极简路由——在「问题处理」**一个**闭合舰员走通，证明三件事：
1. 挡得住「自信翻车」（验收闸 + 副作用闸）；
2. **认证策略复利**——第 2、3 个同类舰员近乎免认证（这是飞轮的活体证据，也是破鸡蛋死锁的钥匙：从第 2 个同类舰员起即显效）；
3. 比手搓 `/loop` 强（手搓能跑不安全/不充分的 Loop，准入层不让）。

**生长**：再加执行器 → 加一个带采纳率闸的 Goal 工厂 → N 变大时路由/地图成形。
**不在第一块砖里**（v2）：满地图、半开放舰员作证（按定义造不出，别安排注定翻车的 demo，P0-E）、工厂全壳。

## §6 为什么这次站得住
- 红队 5 条 P0 **全部补成了机制，不是改名**：认证瓶颈→认证策略+背压；Goal 工厂噪声→采纳率代理闸；假飞轮→认证策略真飞轮；需求设计孤儿→拆解+诚实标注；项管范畴错→认作元层。
- R2 的致命 BREAK 被**真正修复**（认证策略 ≠ 验收模板，不违反 R2 又接回飞轮轴），不是绕过。
- **它就是你要的产品**：全生命周期舰队、Skill→Plugin→Loop→Workbench、Builder 操作它——只是把「做」交给舰员、把「判断」复利成认证策略。
