## 🔴🔴 最高优先级:R-9 不可信内容 → 规则投毒 → 永久后门链(安全 P0)
**这是整场冲刺最重要的发现,且它击穿了 R2 的安全网。**

注入链(四威胁串成一根绳):
1. 竞品雷达抓取攻击者可控的竞品站(产品**常态**信道,零成本)→ 投毒页注入;
2. 注入把一批夹带 **auth 后门**的改动**伪装成同模式低危**;
3. 工程师在 `approve-all-similar` 下批量放行;
4. 批量放行**天然产出一条 auto-approve 规则**(刃B);
5. 规则毕业靠 shadow「人会怎么判」当真值,而该真值**早被同批注入样本带偏**、一致率虚高 → 规则毕业 active;
6. 此后该类 auth 后门**全 fleet 自动放行、无人再看**;
7. 因后门**不触发功能回滚**,§4.6③ 寄望的"下游回滚率兜底"**整条失效** → 后门永久潜伏。

**为何致命**:入口是常态零成本信道;它不是绕过复利机,而是**利用复利机"用人判断当真值"的核心机制反向洗白**,把一次注入放大成永久、规模化、自动执行的后门;专挑下游探测不到的安全后门。

**承重修复(必须解决项)**:
1. **全链路 `trustTier` 标注** —— 区分「可信系统指令」vs「不可信内容」(issue 正文 / 雷达抓取);这是承重墙,T2/T3/T6 都靠它。
2. **data / instruction 隔离** —— 不可信内容永不作为指令进 agent;
3. **规则毕业禁用"被注入源喂养的样本"作真值**;来源不可信的样本不得计入 shadow 一致率;
4. **`Gate.boundary` 锚定 diff 事实,而非 agent 的分类/声明**(防伪装低危);
5. **四眼必须真正独立**(不同信息源/不共享被污染上下文)。
