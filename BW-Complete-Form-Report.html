02multica 研究与概念映射

研究方式：对 multica-ai/multica 做 git clone --depth 1（commit e1d0d68c，2026-07-14），通读 README、 官方全景文档 docs/product-overview.md、LICENSE，以及关键源码 server/internal/daemon/prompt.go（438 行，四类任务提示词全文）与 execenv/ 上下文注入层。本节所有陈述均来自真实抓取的源码，无一来自训练记忆。

multica 是什么

「Your next 10 hires won't be human.」——开源 managed-agents 平台：把 coding agent 变成任务看板上的真实队友。像指派同事一样把 issue 指派给 agent，它自己认领、写码、 发评论、汇报阻塞、更新状态。架构为 Go 后端（Chi + gorilla/websocket + sqlc）+ Next.js 16 前端 + PostgreSQL 17（28 张表）+ 用户本机 daemon（每 3s 轮询认领、15s 心跳、 spawn agent CLI 子进程并流式回传），支持 Claude Code / Codex / Cursor Agent 等 14 种 agent CLI。License 为修改版 Apache 2.0（Dify 式附加条件：未授权不得对第三方 提供托管/嵌入商业服务；组织内部使用免费）——BW 只借鉴概念，零代码复制。

概念映射：multica ↔ builders-workbench
